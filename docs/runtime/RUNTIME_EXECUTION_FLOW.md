# Runtime Execution Flow

## Automation Event Lifecycle

```
EMIT                    FETCH                   PROCESS                  FINALIZE
─────                   ─────                   ───────                  ────────
emitEvent()         →   processor selects   →   optimistic lock      →   update status
  │                     WHERE                   UPDATE SET               completed /
  ├─ dispatchAutomation  processed=false          processing=true         failed /
  │   INSERT INTO        processing=false         status='processing'     dead_letter
  │   automation_events  scheduled_for≤now
  │   status='pending'
  │
  └─ idempotency_key
     prevents duplicate
     inserts via upsert
```

## Event Status FSM

```
            ┌──────────────────────────────────────────┐
            │                                          │
pending ──► processing ──► completed                  │
   ▲            │                                      │
   │            ▼                                      │
   │         failed ──► retrying ──► (re-queued) ──────┘
   │            │
   │            ▼ (attempt_count >= 5)
   └──────── dead_letter
```

Status values match `automation_events.status` CHECK constraint:
`pending | processing | retrying | completed | failed | dead_letter`

## Processor Batch Loop (runAutomationProcessorJob)

1. **Select** up to `batchSize` events WHERE `processed=false AND processing=false AND (scheduled_for IS NULL OR scheduled_for <= now)`
2. **For each event**:
   a. **Lock**: UPDATE SET `processing=true, status='processing'` WHERE `processing=false` (optimistic — skips if lost race)
   b. **Resolve correlation ID**: from payload or `generateCorrelationId("ae")`
   c. **Registry lookup**: `getWorkflowByEventName(event_name)` → workflowId, SLA, queue
   d. **SLA start**: `startSLARecord(workflowId, tenantId, correlationId, eventId)`
   e. **Dispatch**: `processAutomationEvent(id, name, creatorId, payload)`
   f. **Success**: UPDATE `processed=true, status='completed'`, `completeSLARecord(slaId, "completed")`, `logAutomation(..., "success")`
   g. **Failure**: increment `attempt_count`, set `status='retrying'|'failed'|'dead_letter'`, `completeSLARecord(slaId, "failed")`, `logAutomation(..., "failure")`
3. **Return** `{ processed, failed, durationMs }`

## Self-Healing: Stuck Queue Recovery

Every 5 minutes, `runStuckQueueRecoveryJob()` scans for events where:
- `processed=false AND processing=true AND updated_at < (now - 5 min)`

These are reset to `processing=false` with `last_error = "Recovered by self-healing job"`.

This guards against worker crashes that leave events perpetually in-flight.

## Correlation ID Propagation

Every event carries a `correlationId` through its entire lifecycle:

```
emitEvent({ correlationId }) →
  automation_events.payload.correlationId →
    processor extracts →
      processAutomationEvent payload →
        handlers.ts HandlerContext.correlationId →
          logAutomation({ ctx: { correlationId } }) →
            automation_logs.correlation_id
```

Format: `"ae_<timestamp>_<random>"` generated by `generateCorrelationId("ae")`.

## Handler Dispatch

`processAutomationEvent(eventId, eventName, creatorId, payload)` looks up `HANDLERS[eventName]`. Each handler receives `HandlerContext`:

```typescript
interface HandlerContext {
  creatorId:     string
  payload:       Record<string, unknown>
  eventId:       string
  correlationId: string | undefined
  workflowId:    string | undefined
}
```

Unregistered event names return `{ ok: true }` — silently skipped, not an error.
